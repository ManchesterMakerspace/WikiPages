Transactions

Transactions can be reviewed as a whole from Billing Central or on a per-member basis in the member’s profile. In the event of any issues with the software, you may log into Braintree to take similar actions.

 

View and refund any transaction from Billing Central

  1. Login as an admin user and navigate to Billing Central, using the “Billing” option from the upper right hamburger menu
  2. Select the “Transactions” tab to view the transactions table.
  3. Transactions are filterable by date and by member as indicated in the image below
  4. To refund a transaction, select the row, click “Refund Transaction” button and accept the prompt. NOTE: Only Paid transactions can be refunded. Pending transactions can be voided through Braintree as described below.

View and refund a member’s transaction from their profile

  1. Login as a admin user and navigate to a member’s profile, then select “Payment History” to view their transactions
  2. These transactions are also filterable by date.
  3. Select a transaction to refund, click “Refund Transaction” button and accept the prompt. NOTE: Only Paid transactions can be refunded.  Pending transactions can be voided through Braintree as described below.

Voiding a transaction

If a member has just made a payment and realized it was in error, the transaction can be voided from our side before it clears.  Once a payment clears ("settled"), one must use the refund process described above. 

  1. Login to Braintree Gateway using the Makerspace PayPal credentials.
  2. Navigate to "Transactions" by clicking the associated link in the header.
  3. Search for the transaction you would like to void. You can see all transactions that can be voided by limiting the Status property of search to "Authorized", "Submitted For Settlement", and "Settlement Pending".
  4. In the resulting search table, click the associated transaction ID to view the transaction details.
  5. Click the red "Void" button in the upper left hand corner to void the transaction.

Failed Transactions

When we receive a transaction, we will get a Slack notification in #member-relations regardless of success or failure. The related member will also receive a Slack notification privately. In the event of a failure, the reason for failure will also be included in the message. If you are unsure what this failure status means, you can review all transaction status definitions here. Details regarding how to handle failed Subscription payments may be found here. If you are worried about how many transactions get declined, please note the industry standard is about 10%.

Manually Retry Transactions

  1. Login to Braintree Gateway using the Makerspace PayPal credentials.
  2. Navigate to "Subscriptions" by clicking the associated link in the header.
  3. Search for the subscription you would like to retry. You can see all subscriptions that can be retried by limiting the Status property of search to "past due".
  4. In the resulting search table, click the "Retry" link in the right side of the table.
  5. Verify the transaction details and create a new transaction.

Changing Payment Method

  1. Payment methods cannot be changed for failed transactions unless it is a failed subscription transaction.  If the payment method is invalid for a one off transaction, the proper way to change the Payment Method is to void the failed transaction and submit a new one.
